# 二、介绍

## 2.1 Multithreading
GStreamer是一个多线程框架。它可以根据自己的需求创建或者销毁线程。

在此基础上，当构建管道时，应用程序可以显式地指定分支(管道的一部分)在不同的线程上运行(例如，让音频和视频解码器同时执行)

这是使用队列元素完成的，它的工作方式如下。sink pad只是对数据进行排队并返回控制。在另一个线程中，数据被排出队列并被推到下游。
